### 📊 النظام المحاسبي المحدث

#### دين السائق:
```
دين السائق = (الأجرة المتوقعة - الأجرة المدفوعة)
            + (سداد مخالفة + سداد رسوم إقامة + سداد رسوم رواتب)
            - (تحصيل مخالفة + تحصيل رسوم إقامة + دين قديم)
            + الديون القديمة المسجلة

⚠️ ملاحظة: تحصيل رسوم الرواتب وتحصيل رسوم إيداعات الرواتب لا تؤثر على الدين
```

#### الإيرادات:
```
إجمالي الإيرادات = الإيرادات المباشرة
                   + (أجرة يومية + تحصيل مخالفة + تحصيل رسوم إقامة 
                      + تحصيل رسوم رواتب + تحصيل رسوم إيداعات الرواتب + دين قديم)
```

#### الرصيد البنكي:
```
الرصيد البنكي = الإيرادات 
                - المصروفات (بما فيها مصاريف إيداعات الرواتب)
                + (أجرة يومية + تحصيل مخالفة + تحصيل رسوم إقامة 
                   + تحصيل رسوم رواتب + تحصيل رسوم إيداعات الرواتب + دين قديم)
                - (سداد مخالفة + سداد رسوم إقامة)
```

#### رصيد الرواتب:
```
رصيد الرواتب = (تحصيل رسوم رواتب + تحصيل رسوم إيداعات الرواتب)
               - (سداد رواتب + مصاريف إيداعات الرواتب)
```

## ✅ التعديل السادس: إضافة نظام الأرقام المرجعية التسلسلية (refNum)

**📅 التاريخ**: 1 أبريل 2026  
**Commit ID**: `7f92f13`

### 📋 الهدف
إضافة رقم مرجعي تسلسلي فريد لكل إيراد ومصروف ودفعة سائق، لتسهيل الرجوع لأي سجل في أي وقت.

### 🎯 تفاصيل التصميم

**صيغة الرقم المرجعي**: `PREFIX + YYYYMMDD + NNN` (بدون فواصل)

| النوع | البادئة | مثال |
|-------|---------|------|
| إيراد مباشر | `REV` | `REV20260401001` |
| مصروف | `EXP` | `EXP20260401001` |
| دفعة سائق | `PAY` | `PAY20260401001` |

- الرقم التسلسلي يُعاد من `001` كل يوم جديد
- كل نوع مستقل عن الآخر (إيرادات مستقلة، مصروفات مستقلة، دفعات مستقلة)

### ✨ التعديلات المطبقة

## 🔧 إصلاح: عدم ظهور الرقم المرجعي عند أول تحميل صفحة الإيرادات

**📅 التاريخ**: 1 أبريل 2026  
**Commit ID**: `96b96b1`

### 🎯 المشكلة
عند أول تحميل صفحة الإيرادات، كان الرقم المرجعي لا يظهر في عمود دفعات السائقين. لكن بعد استخدام أي فلتر (يوم/أسبوع/كل) كان يظهر بشكل صحيح.

### 🔍 السبب الجذري
دالة `displayRevenuesWithDriverPayments()` في `revenues.html` كانت تبني قائمة دفعات السائقين **بدون نقل حقل `refNum`**:

```javascript
// ❌ قبل الإصلاح - refNum غائب
allRevenueItems.push({
    id: payment.id,
    date: payment.date,
    type: payment.type,
    amount: payment.amount,
    description: ...,
    driverId: payment.driverId,
    driverName: ...,
    employeeNumber: ...,
    source: 'driverPayment'
    // refNum مفقود!
});
```

أما دالة `applyFilters()` (التي تُستدعى عند الضغط على الفلتر) كانت تضيف `refNum` بشكل صحيح، لذلك كان يظهر بعد استخدام الفلتر فقط.

### ✅ الحل
إضافة `refNum: payment.refNum || null` في دالة `displayRevenuesWithDriverPayments()`:

```javascript
// ✅ بعد الإصلاح
allRevenueItems.push({
    id: payment.id,
    date: payment.date,
    type: payment.type,
    amount: payment.amount,
    description: ...,
    driverId: payment.driverId,
    driverName: ...,
    employeeNumber: ...,
    refNum: payment.refNum || null,  // ✅ تم الإضافة
    source: 'driverPayment'
});
```
